![e3002578-691d-4d54-a16f-13b522fd3c6f](https://github.com/user-attachments/assets/49f11bed-fd49-4ac5-9652-1106adfc3156)

Сервис для сокращения ссылок на **Go** с использованием **PostgreSQL**, **Docker** и **REST API**. Проект создавался для практики языка, популярных сторонних пакетов, разных видов тестирования.
Взаимодействие происходит через запросы на http-server. Ссылки хранятся в базе данных PostgreSQL.

## Расширенное описание
### API Endpoints

Базовый URL: `http://localhost:8080/v1`

| Метод  | Эндпоинт | Описание                                                   | Пример запроса                                                      | Пример ответа                                                      |
| ------ | -------- | ---------------------------------------------------------- | ------------------------------------------------------------------- | ------------------------------------------------------------------ |
| POST   | `/`      | Создать короткую ссылку из оригинальной                    | `{ "url": "https://example.com" }`                                  | `{ "url": "https://click.ru/AB1cDE" }`                             |
| GET    | `/short` | Получить короткую ссылку по оригинальной                   | `?orig=https://example.com`                                         | `{ "url": "https://click.ru/AB1cDE" }`                             |
| GET    | `/orig`  | Получить оригинальную ссылку по короткой                   | `?short=https://click.ru/AB1cDE`                                    | `{ "url": "https://example.com" }`                                 |
| GET    | `/url`   | Получить дополнительную информацию ссылки                  | `?search=https://click.ru/AB1cDE` или `?search=https://example.com` | `{ "orig_url": "...", "short_url": "...", "created_at": "..." }`   |
| GET    | `/urls`  | Получить список дополнительной информации ссылок с лимитом | `?limit=10`                                                         | `[{ "orig_url": "...", "short_url": "...", "created_at": "..." }]` |
| DELETE | ...      | Удалить ссылку (не реализовано)                            | –                                                                   | –                                                                  |

### Генерация короткой ссылки
Алгоритм генерирует короткую ссылку фиксированной длины.
Генерация происходит на основе переданного аргумента `counter`.

1. **Принцип работы**: короткая ссылка формируется путём последовательного деления числового идентификатора на основания `26, 26, 9, 26, 26, 26` с использованием символов `A–Z`, `a–z` и `1–9`. Последние 6 символов строки `https://click.ru/` заменяются на полученные символы.
2. **Гарантии уникальности**: алгоритм обеспечивает уникальные короткие ссылки для первых **106 932 384** значений счётчика (от 0 до 106 932 383). При превышении этого предела уникальность не гарантируется — возможны повторения.
3. **Ограничения**: внутренняя логика вызовет панику при достижении максимального значения уникальности.

```markdown
Максимальное количество уникальных ссылок:  
26 × 26 × 9 × 26 × 26 × 26 = 26⁵ × 9 = 106 932 384
```
### База данных
Выбрана база данных PostgreSQL.
Определены две таблицы counter и urls также Hash индексы для orig_url, short_url.

#### Таблица `counter`
| Поле      | Тип           | Ограничения                |
|-----------|---------------|----------------------------|
| id        | INT           | PRIMARY KEY                |
| counter   | BIGINT        | NOT NULL, DEFAULT 0        |

#### Таблица `urls`
| Поле        | Тип        | Ограничения                |
|-------------|------------|----------------------------|
| orig_url    | TEXT       | PRIMARY KEY                |
| short_url   | TEXT       | NOT NULL                   |
| created_at  | TIMESTAMP  | NOT NULL                   |

### Тесты
Были реализованы 3 вида тестов:
1. **Unit тесты**
	- Тестируют каждый пакет. 
	- Определяются для каждого пакета отдельно.
2. **Integration тесты**
	- Тестируют взаимодействие с внешними зависимостями, конкретно с базой данных.
	- База данных подменялась на тестовую копию.
	- Каждый тест откатывает базу данных до первоначального состояния.
3. **End To End тесты**
	- Тестирует поведение приложения с позиции клиента.
	- В тестах определяется клиент и от него производятся запросы.
	- База данных подменялась на тестовую копию.
	- Каждый тест создает новый http-server, откатывает базу данных до первоначального состояния.

## How to use
Очень просто! Выполняй http запрос на Endpoints.

Базовый адрес: `http://localhost:8080/v1`  
Все запросы и ответы в формате JSON.
### Создание короткой ссылки

**POST** `/`  
```bash
curl -X POST http://localhost:8080/v1/ \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com"}'
```

**Ответ (200 OK):**
```json
{
  "url": "https://click.ru/abc123"
}
```
### Получение оригинальной ссылки по короткой

**GET** `/shorturl` (параметр в query)  
```bash
curl -X GET "http://localhost:8080/v1/orig?short=https://click.ru/abc123"
```

**Ответ (200 OK):**
```json
{
  "url": "https://example.com"
}
```
